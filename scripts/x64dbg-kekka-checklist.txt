KekkaPlayer createWindow crash — x64dbg checklist
=================================================

1) RIGHT THREAD
   View -> Threads (or Alt+T)
   Select thread named: API thread
   If you are on GC Thread / AWT-EventQueue — switch to API thread.

2) REGISTERS (Register tab, or type: r)
   RAX = 0        -> null pointer used as object (Flash/COM not created)
   RSI = 0        -> same, often "this" for COM call
   RIP in jvm.dll -> JNI stub calling native KekkaPlayer
   Exception: C0000005 READ at address 0

3) CALL STACK (Stack tab, or type: k / kp)
   Top should look like:
     jvm.dll+0x3e9027          (JNI invoke)
     KekkaPlayer.dll+0x36287   (createWindow native — crash site)
     eu.darkbot.api.KekkaPlayer.createWindow
     KekkaPlayerLauncher / API thread

4) MODULES
   Confirm KekkaPlayer.dll is loaded from:
     ...\net10.0\lib\KekkaPlayer.dll
   Note base address + 0x36287 = exact crash offset (changes each run).

5) OPTIONAL — break earlier in createWindow
   After KekkaPlayer.dll loads (Modules list):
     bp KekkaPlayer.dll+36287
   Then run (F9). When it breaks BEFORE crash, step (F7/F8) and watch which
   call uses RAX/RCX when they become 0.

6) WHAT IT MEANS
   setFlashOcxPath / setData return OK in Java log, but createWindow dies
   reading null -> internal Flash ActiveX pointer never allocated.
   Compare with working Java DarkBot: setupAuth() runs before KekkaPlayer loads.

Commands in x64dbg command bar:
  r          — registers
  k          — stack
  mod        — module list (find KekkaPlayer base)
  erun       — run, pass first-chance exceptions
  bca        — clear all breakpoints
